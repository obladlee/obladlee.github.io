Utility Network 公共设施网络

2020-07-06 Utility Network

文章目录

  1. 建模
  2. 可视化
  3. 分析
  4. 编辑
  5. 参考

最近接手了一项研究Utility Network的任务，译为公共设施网络。它是用户在ArcGIS中管理公共设施和通信网络时使用的主要组件，提供了综合框架，用于对电力、天然气、水利、雨水、废水和通信等公共设施系统进行建模。能够对系统内的每一个组成部分进行建模-例如电线、管道、阀门、区域、设备和回路，同时建模要素之间的真实逻辑行为。依目前的理解，UN有四大功能，建模、可视化、编辑、分析。预计UN的研究会是一个长篇系列，在此也给自己挖个坑，希望以后能一步步完善。

建模

UN能够建模现实网络中的要素与逻辑行为。使用UN，可以进行以下操作：

  • 创建和编辑每种公共设施设备的建模要素。
  • 发现要素在网络内的连接方式。
  • 追踪资源，例如天然气、水和电力在网络中的流动。
  • 提供显示公共设施所有动态设备当前配置状况的操作视图。
  • 分析现实世界事件，例如风暴、停电或设备故障对网络造成的影响。

访问UN有单用户与多用户模型之分：单用户模型基于fgdb建立，可以再ArcGIS Pro中与UN交互。多用户模型基于ArcGIS Enterprise和egdb建立，可在web、移动端、桌面端与UN交互。

可视化

可视化是基础功能，UN提供了多种方式查看网络系统与资产，比如专题制图地图、逻辑示意图等。

分析

UN提供了分析与追踪工具，支持多种分析工作流，比如从给定位置向上游或下游追踪网络要素。例如，自来水公司可确定在管道破裂时要关闭哪些水阀。

编辑

编辑UN也是基础功能，可以基于模板创建各种公共资产，多用户模型允许多位用户基于服务的架构并发管理编辑要素。

UN的复杂性在于对现实网络设施、以及资产和网络之间的逻辑关系的建模，数据越多越复杂，UN复杂度也越高。Esri从五六年前开始和行业解决方案提供商合作开发的Utility Network，其商用性有很大的应用前景。
